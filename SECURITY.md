# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in TLGMapper, please report it responsibly by emailing the maintainers at [security@tlgmapper.dev](mailto:security@tlgmapper.dev) (placeholder - replace with actual contact).

We take security issues seriously and will respond promptly to valid reports. Please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested remediation approaches

## Scope

Security reports should focus on vulnerabilities in:

- TLGMapper's provider parsing logic
- Validation routines
- Binary extraction methodology
- CI pipeline security
- Dependency-related risks
- Repository integrity mechanisms

## Out of Scope

The following are explicitly out of scope for security reports:

- Windows kernel behavior or ETW system internals
- Third-party dependencies (report directly to the respective maintainers)
- General Windows security issues not specific to TLGMapper
- Performance issues without security impact

## Response Process

1. **Acknowledgment:** We will acknowledge receipt of your report within 48 hours
2. **Investigation:** We will investigate the issue and provide regular updates
3. **Remediation:** We will work to develop and release a fix
4. **Disclosure:** We will coordinate disclosure timing with you

## Safe Harbor

We commit to not pursuing legal action against researchers who report security vulnerabilities in good faith, provided they follow this disclosure policy and do not engage in:

- Unauthorized access to systems
- Data destruction or modification
- Service disruption
- Public disclosure before remediation

## Contact

For security-related inquiries, please use the dedicated security contact above rather than public issues or discussions.
